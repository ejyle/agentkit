    Create cmd/doctor.go with package cmd. Define a CheckResult struct with fields: Label string, Status string (values: "pass", "warn", "fail"), Message string, Hint string.

    Define doctorCmd as a *cobra.Command with Use: "doctor", Short: "Check your agentkit environment", RunE: runDoctor. The RunE function must NOT be set on a persistent pre-run that validates --target, because doctor does not take a target flag. Override PersistentPreRunE to nil on doctorCmd to bypass the root command's target validation: set doctorCmd.PersistentPreRunE = func(...) error { return nil } in init().

    Implement runDoctor(cmd *cobra.Command, args []string) error that:
    1. Collects results by calling check functions in D-08 order
    2. Prints each result with printCheckResult()
    3. If any result has Status "fail", returns fmt.Errorf("one or more checks failed") — Cobra will set exit code 1

    Implement these check functions (each returns CheckResult or []CheckResult):

    checkBinaryInPath() CheckResult
    - Use os/exec.LookPath("agentkit")
    - On success: Label "agentkit in PATH", Status "pass", Message "(version from os.Args[0] or binary path)"
    - On failure: Status "fail", Hint "Add agentkit to your PATH"

    checkConfigDirWritable() CheckResult
    - Dir: filepath.Join(home, ".agentkit") where home comes from os.UserHomeDir()
    - Call os.MkdirAll(dir, 0755); if err → Status "fail", Hint "Run: mkdir -p ~/.agentkit"
    - Write a temp file filepath.Join(dir, ".write-test"), os.Remove it; if err → Status "fail"
    - On success: Label "~/.agentkit/ writable", Status "pass"

    checkRegistryReachable() CheckResult
    - URL: https://raw.githubusercontent.com/ejyle/agentkit-registry/main/registry.json
    - Use context.WithTimeout(context.Background(), 5*time.Second) and http.NewRequestWithContext with method HEAD
    - Use http.DefaultClient.Do(req); check err or resp.StatusCode >= 500
    - On success: Label "registry reachable (agentkit-registry)", Status "pass"
    - On failure: Status "fail", Hint "Check network connectivity"

    checkAssistantDirs() []CheckResult
    - Check these 5 dirs: ~/.claude/, ~/.gemini/, ~/.copilot/, ~/.codex/, ~/.config/opencode/
    - Use os.Stat() to test existence
    - On exists: Status "pass", Label "~/.claude/ exists" (etc.)
    - On not exists: Status "warn" (NOT fail — per D-08/D-09), Message "not found — <AssistantName> not installed"
    - No Hint for missing assistant dirs (user may not use that assistant)

    checkRuntimeDeps() []CheckResult
    - Check three binaries: "node", "docker", "uvx" using exec.LookPath
    - All three: Status "warn" (NOT fail) when missing
    - node missing: Label "node", Message "not found — npx-based MCPs won't install", Hint "Install: https://nodejs.org"
    - docker missing: Label "docker", Message "not found — Docker-based MCPs won't install", Hint "Install: https://docs.docker.com/get-docker/"
    - uvx missing: Label "uvx", Message "not found — Python MCPs won't install", Hint "Install: pip install uv"
    - On found: Status "pass", Label "node available" (etc.)

    Implement printCheckResult(r CheckResult):
    - "pass" → fmt.Printf("✓ %s\n", r.Label) — or include r.Message if non-empty
    - "warn" → fmt.Printf("⚠ %s", r.Label); if r.Message non-empty print " — " + r.Message; println; if r.Hint non-empty print "   → " + r.Hint + newline
    - "fail" → fmt.Fprintf(os.Stderr, "✗ %s", r.Label); same pattern for Message and Hint but to stderr

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| exec.LookPath calls | User-controlled PATH; only reads, never executes the found binary |
| HTTP network check | Outbound to raw.githubusercontent.com; no user-supplied URL |
| Filesystem write test | Writes to ~/.agentkit/.write-test only; always cleaned up |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02-01 | Tampering | ~/.agentkit/.write-test | accept | Temp file is created and immediately removed; 0-byte write; no data exposure |
| T-04-02-02 | Denial of Service | checkRegistryReachable | mitigate | context.WithTimeout(ctx, 5*time.Second) hard caps network wait; never use http.Get() |
| T-04-02-03 | Information Disclosure | ✗ lines printed to stderr | accept | Only reports existence of dirs/binaries; no secrets printed |
| T-04-02-04 | Elevation of Privilege | os.MkdirAll ~/.agentkit | accept | Creates user-owned dir under $HOME with 0755; no sudo required; no escalation possible |
| T-04-02-SC | Tampering | npm/pip/cargo installs | accept | No package manager installs in this plan; go.mod unchanged |
</threat_model>
