<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GoReleaser build | Reads go.sum-pinned source; CGO_ENABLED=0 ensures no dynamic linking |
| HOMEBREW_TAP_GITHUB_TOKEN | Secret PAT crosses trust boundary from ejyle/agentkit CI to ejyle/homebrew-agentkit |
| cosign OIDC | Ephemeral key from GitHub OIDC — no static secret stored |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03-01 | Tampering | go.sum supply chain | mitigate | CGO_ENABLED=0 eliminates libc linkage; go.sum pins all module hashes; GoReleaser builds from clean checkout |
| T-04-03-02 | Information Disclosure | HOMEBREW_TAP_GITHUB_TOKEN | mitigate | Stored as GitHub Actions secret (never in logs); scoped to repo scope on tap repo only; not accessible to PRs from forks |
| T-04-03-03 | Tampering | Malicious tap push via leaked PAT | mitigate | PAT has repo scope on homebrew-agentkit only; cannot modify ejyle/agentkit source; cask formula is generated by GoReleaser from verified checksums |
| T-04-03-04 | Tampering | Tampered binary in GitHub Release | mitigate | cosign keyless signing produces checksums.txt.sigstore.json; users can verify with cosign verify-blob |
| T-04-03-SC | Tampering | npm/pip/cargo installs | accept | No package manager installs; GoReleaser is a CI tool installed by goreleaser-action, not go get |
</threat_model>
