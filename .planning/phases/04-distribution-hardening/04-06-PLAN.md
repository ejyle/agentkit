<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Tag push triggering CI | Only maintainers with push access can push v* tags |
| GitHub Release assets | Users download from GitHub CDN over TLS; cosign bundle allows independent verification |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-06-01 | Tampering | Release tag pushed before code is ready | mitigate | Pre-flight checks (go build, go vet, local doctor run) gate the tag push |
| T-04-06-02 | Tampering | User downloads tampered binary | mitigate | checksums.txt.sigstore.json enables cosign verify-blob; SHA256 in install.sh covers install script path |
| T-04-06-03 | Spoofing | Impersonating v0.1.0 release | accept | GitHub OIDC ephemeral certificate in sigstore bundle ties release to the specific workflow run and repo |
| T-04-06-SC | Tampering | npm/pip/cargo installs | accept | No package manager installs in this plan |
</threat_model>
