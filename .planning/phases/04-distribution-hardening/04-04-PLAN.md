    Workflow name: "Release". on: trigger with two events: push.tags matching ["v*"] AND push.branches matching ["main"].

    Define two jobs: "release" and "snapshot".

    release job:
    - if: startsWith(github.ref, 'refs/tags/')
    - runs-on: ubuntu-latest
    - permissions: contents: write, id-token: write  (id-token: write is MANDATORY for cosign OIDC — do not omit)
    - steps:
      1. actions/checkout@v4 with fetch-depth: 0  (MANDATORY — GoReleaser needs full history for changelog; shallow clone breaks it)
      2. actions/setup-go@v5 with go-version: stable
      3. sigstore/cosign-installer@v3  (installs cosign binary; no with params needed — installs latest)
      4. goreleaser/goreleaser-action@v7 with:
           distribution: goreleaser
           version: "~> v2"
           args: release --clean
         env:
           GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
           HOMEBREW_TAP_GITHUB_TOKEN: ${{ secrets.HOMEBREW_TAP_GITHUB_TOKEN }}

    snapshot job:
    - if: github.ref == 'refs/heads/main'
    - runs-on: ubuntu-latest
    - permissions: contents: read  (snapshot does not upload to GitHub Releases)
    - steps:
      1. actions/checkout@v4 with fetch-depth: 0
      2. actions/setup-go@v5 with go-version: stable
      3. goreleaser/goreleaser-action@v7 with:
           distribution: goreleaser
           version: "~> v2"
           args: release --snapshot --clean --skip=publish,sign,homebrew
         env:
           GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}

    Do NOT add cosign-installer to the snapshot job — signing is skipped via --skip=sign.
    Do NOT add HOMEBREW_TAP_GITHUB_TOKEN to the snapshot job — homebrew is skipped.
    Do NOT use goreleaser-action@v5 or @v6 — use @v7.
    Do NOT set COSIGN_EXPERIMENTAL=1 — not needed in cosign v3+ / v1.13+.
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions secrets → workflow env | GITHUB_TOKEN auto-scoped; HOMEBREW_TAP_GITHUB_TOKEN read from repo secrets; never logged |
| PR from fork | Fork PRs do not have access to repo secrets — snapshot job would skip publish anyway |
| id-token: write permission | Grants OIDC token to the workflow job for cosign; scoped to this job only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04-01 | Information Disclosure | HOMEBREW_TAP_GITHUB_TOKEN in workflow | mitigate | Secret referenced via ${{ secrets.* }} — GitHub masks in logs; never interpolated into shell |
| T-04-04-02 | Tampering | Fork PR triggering release job | accept | release job only runs on refs/tags/* — fork PRs cannot push tags to origin |
| T-04-04-03 | Spoofing | OIDC identity for cosign | accept | GitHub OIDC ties identity to repo + workflow run; ephemeral cert expires after signing |
| T-04-04-04 | Tampering | Snapshot publishing to tap | mitigate | --skip=publish,sign,homebrew on snapshot job; no tap pollution |
| T-04-04-SC | Tampering | npm/pip/cargo installs | accept | No package manager installs; all Actions are from official orgs with pinned major versions |
</threat_model>
