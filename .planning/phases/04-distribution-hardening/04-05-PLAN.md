  <action>
    Create the scripts/ directory if it does not exist, then create scripts/install.sh.

    Start with shebang: #!/usr/bin/env sh and set -euo pipefail (POSIX-compatible).

    VERSION variable: default to the latest release. Use: VERSION="${AGENTKIT_VERSION:-0.1.0}" so the user can override with AGENTKIT_VERSION=0.2.0 sh install.sh.

    OS detection: OS=$(uname -s | tr '[:upper:]' '[:lower:]'). Then case on $OS: linux → OS="linux", EXT="tar.gz"; darwin → OS="darwin", EXT="tar.gz"; default → echo "Unsupported OS: $OS" >&2; exit 1.

    ARCH detection: ARCH=$(uname -m). Then case on $ARCH: x86_64 → ARCH="amd64"; aarch64 | arm64 → ARCH="arm64"; default → echo "Unsupported architecture: $ARCH" >&2; exit 1.

    Checksum tool detection: if command -v sha256sum >/dev/null 2>&1; then SHA_CMD="sha256sum"; elif command -v shasum >/dev/null 2>&1; then SHA_CMD="shasum -a 256"; else echo "Cannot verify checksum — neither sha256sum nor shasum found" >&2; exit 1; fi.

    FILENAME: agentkit_${VERSION}_${OS}_${ARCH}.${EXT}
    BASE_URL: https://github.com/ejyle/agentkit/releases/download/v${VERSION}

    Download to TMPDIR (use mktemp -d for a clean temp dir, not hardcoded /tmp):
    - TMPDIR=$(mktemp -d)
    - trap 'rm -rf "$TMPDIR"' EXIT  (clean up on exit, success or failure)
    - Download archive: curl -fsSL "${BASE_URL}/${FILENAME}" -o "${TMPDIR}/${FILENAME}"
    - Download checksums: curl -fsSL "${BASE_URL}/checksums.txt" -o "${TMPDIR}/checksums.txt"

    Checksum verification (MUST happen before any binary execution):
    - cd "${TMPDIR}"
    - Run ${SHA_CMD} --check --ignore-missing checksums.txt (Linux sha256sum syntax)
    - Note: shasum -a 256 uses the same --check --ignore-missing flags
    - If this fails, the script exits immediately due to set -e

    Extract and install:
    - tar -xzf "${TMPDIR}/${FILENAME}" -C "${TMPDIR}" agentkit
    - INSTALL_DIR="${HOME}/.local/bin"
    - mkdir -p "${INSTALL_DIR}"
    - mv "${TMPDIR}/agentkit" "${INSTALL_DIR}/agentkit"
    - chmod +x "${INSTALL_DIR}/agentkit"

    Print success and PATH hint:
    - printf 'Installed agentkit %s to %s\n' "${VERSION}" "${INSTALL_DIR}/agentkit"
    - printf 'Add to PATH: export PATH="$HOME/.local/bin:$PATH"\n'

    After creating the file, make it executable: chmod +x scripts/install.sh.

    Do NOT hardcode /tmp — use mktemp -d.
    Do NOT skip the checksum step or make it optional.
    Do NOT use wget — use curl -fsSL only.
    Do NOT install to /usr/local/bin (would require sudo) — ~/.local/bin only.
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| curl download | Downloads from github.com/ejyle/agentkit/releases — GitHub CDN; TLS-protected |
| Checksum verification | SHA256 from GoReleaser's checksums.txt verifies archive integrity before binary execution |
| User PATH | Script installs to ~/.local/bin — user scope, no sudo |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05-01 | Tampering | Tampered binary download | mitigate | SHA256 checksum verified against GoReleaser-generated checksums.txt BEFORE install; set -e exits on mismatch |
| T-04-05-02 | Spoofing | MITM on curl download | mitigate | curl -fsSL enforces TLS; GitHub certificates; SHA256 post-verification catches any tampering |
| T-04-05-03 | Tampering | Malicious AGENTKIT_VERSION override | accept | User can only install versions that exist on GitHub Releases; invalid version = 404 from curl -f |
| T-04-05-04 | Elevation of Privilege | Install destination | accept | Installs to ~/.local/bin only; no sudo; cannot write to system paths |
| T-04-05-05 | Denial of Service | Temp dir pollution | mitigate | trap 'rm -rf "$TMPDIR"' EXIT ensures cleanup on success or failure |
| T-04-05-SC | Tampering | npm/pip/cargo installs | accept | Shell script only; no package manager invocations |
</threat_model>
