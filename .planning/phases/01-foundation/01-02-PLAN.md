<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| registry client → GitHub raw CDN | HTTPS fetch of registry.json; MITM risk if cert pinning absent |
| cache write → ~/.cache/agentkit/ | Disk write of fetched manifest; path traversal risk if registry ID not sanitized |
| config store → ~/.config/agentkit/ | Atomic write of installed state; partial write risk mitigated by renameio |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | GitHubManifestRegistry manifest fetch | mitigate | HTTPS-only manifest URLs; Go's net/http validates TLS certificates by default |
| T-02-02 | Tampering | CachedManifest on disk | accept | Cache is advisory only; stale manifest causes stale search results, not code execution |
| T-02-03 | Information Disclosure | go-retryablehttp URL logging | mitigate | Pin go-retryablehttp@v0.7.8+ (CVE-2024-6104 URL sanitization fix) |
| T-02-04 | Tampering | ConfigStore atomic write | mitigate | renameio.WriteFile used for all writes; non-atomic os.WriteFile is prohibited |
| T-02-05 | Tampering | Registry ID used as cache directory name | mitigate | Sanitize registry ID: allow only [a-zA-Z0-9_-] before using in filepath.Join |
| T-02-SC | Tampering | go get installs (this plan adds no new deps) | accept | All deps already vetted in 01-01; no new go get in this plan |
</threat_model>
