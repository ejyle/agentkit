<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user input → exec.Command | Package name from CLI args passed to npx; shell injection risk |
| network → BinaryInstaller | Downloaded binary from registry URL; malicious binary risk |
| ~/.claude.json | User's MCP config; clobbering risk if merge is non-atomic or non-selective |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | NpxInstaller exec.Command | mitigate | Use exec.Command(binary, arg1, arg2) arg-array form; never fmt.Sprintf into shell string. Validate package name matches [a-zA-Z0-9@/._-]+ before exec. |
| T-03-02 | Tampering | BinaryInstaller download | mitigate | SHA256 checksum verification required; mismatch returns ErrChecksumMismatch and aborts install |
| T-03-03 | Spoofing | BinaryInstaller URL scheme | mitigate | Reject any spec.URL with scheme != "https"; return ErrInsecureURL |
| T-03-04 | Tampering | ~/.claude.json merge | mitigate | Read full file as map[string]interface{}, modify only mcpServers key, atomic write via renameio |
| T-03-05 | Tampering | ~/.claude.json foreign conflict | mitigate | D-07: check installed.json ownership before any overwrite; prompt user on conflict |
| T-03-06 | Tampering | Post-install verify | mitigate | MCP-06: re-read config after write; fail loudly if key absent or file unparseable |
| T-03-SC | Tampering | npm/go package installs | accept | No new go get in this plan; all packages vetted in 01-01 |
</threat_model>
