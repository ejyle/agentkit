```toml
[mcp_servers.my-server]
command = "uvx"
args = ["mcp-server-fetch"]

[mcp_servers.my-server.env]
MY_KEY = "value"
```
Config path: ~/.codex/config.toml
Read: toml.DecodeFile into map[string]interface{} (preserves all non-mcp_servers keys)
Write: modify only mcp_servers subtable; bytes.Buffer + toml.NewEncoder; renameio.WriteFile

OpenCode config format (from RESEARCH.md):
```json
{
  "mcp": {
    "my-server": {
      "type": "local",
      "command": ["uvx", "mcp-server-fetch"],
      "environment": {"KEY": "value"},
      "enabled": true
    }
  }
}
```
Config path: os.UserConfigDir() + "/opencode/opencode.json"
Key: "mcp" (not "mcpServers")
command field: JSON array (command + args combined); NOT split command+args
env key: "environment" (not "env")
Cannot embed jsonMCPAdapter (different schema — would require overriding everything)

| manifest → OpenCodeAdapter | command array built from entry.Command + entry.Args; must remain as array (no shell string) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-10 | Tampering | CodexAdapter TOML write — mcp_servers key modification | mitigate | toml.DecodeFile into map → modify only mcp_servers subtable → re-encode; other TOML keys (model, theme, etc.) preserved; test TestCodexAdapter_WriteMCPConfig_PreservesNonMCPKeys validates this |
| T-02-11 | Tampering | OpenCodeAdapter command array construction | mitigate | command built via append([]string{entry.Command}, entry.Args...) — each element is a separate JSON array element; never concatenated into a shell string |
| T-02-12 | Tampering | OpenCodeAdapter "environment" vs "env" key confusion | mitigate | Write uses "environment" key explicitly; ReadMCPConfig reads from "environment" key; test TestOpenCodeAdapter_WriteMCPConfig_EnvironmentKey validates the correct key name |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No new dependencies in this plan |
