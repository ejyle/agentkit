<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| installer → GitHub HTTPS | Tarball fetched from github.com over HTTPS; content from an external repo outside our control |
| archive → filesystem | Tar header names from the tarball are used to construct filesystem paths; must be bounded to destDir |
| registry → installer | InstallSpec.Repo and InstallSpec.Path come from the registry manifest; a compromised registry could inject a malicious repo or path |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | tar entry path (zip-slip) | mitigate | Reject entries where rel contains ".."; also verify filepath.Join(destDir, rel) starts with destDir+sep |
| T-03-02 | Spoofing | GitHub tarball HTTPS | mitigate | HTTPS-only URL enforcement in Install() (same as BinaryInstaller ErrInsecureURL pattern) |
| T-03-03 | Tampering | Disk tarball cache | accept | Cache is under UserCacheDir (user-owned); no cross-user threat; HTTPS already protects download integrity |
| T-03-04 | Tampering | InstallSpec.Repo | mitigate | Registry ID is already sanitized to [a-zA-Z0-9_-] (established in Phase 1 T-02-05); Repo field validated non-empty |
| T-03-05 | Tampering | Extracted SKILL.md skill overwrite | mitigate | Files extracted to SkillInstallPath(target, name) only; bounded by switch in SkillInstallPath rejecting non-skill targets |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No new package manager installs in this plan; all deps are stdlib or already in go.mod |
</threat_model>
