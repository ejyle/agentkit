```yaml
---
name: <skill-name>       # matches folder name; lowercase + hyphens; max 64 chars

---
```

Body rules:
- Markdown; maximum 500 lines
- First section: "## When to Use" — activation triggers for the AI agent
- Second section: "## Quick Reference" — most common commands by sub-domain
- Third section: "## Reference Files" — lists which references/*.md file to load for which task
- Heavy content (full command tables, patterns, config snippets) belongs in references/ not SKILL.md body
- No hardcoded versions in command examples (use "latest" or omit version)

Reference file rules:
- Each references/*.md file: a focused domain reference for a single sub-topic
- Structure: ## Overview (2-3 sentences), ## Common Commands, ## Patterns, ## Gotchas
- Token budget: target 200-400 lines per reference file; technical but concise
