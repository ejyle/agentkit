## Overview

Amazon S3 (Simple Storage Service) stores objects in globally-named buckets. Each object has a key (path), metadata, and optional version ID. Access is controlled through bucket policies, IAM policies, and ACLs. The AWS CLI `aws s3` (high-level) and `aws s3api` (low-level API) cover all operations.

## Common Commands

### Bucket Operations

```bash
# Create bucket (us-east-1 does not need LocationConstraint)
aws s3api create-bucket --bucket my-unique-bucket-name --region us-east-1

# Create bucket in other regions
aws s3api create-bucket --bucket my-bucket --region eu-west-1 \
  --create-bucket-configuration LocationConstraint=eu-west-1

# List buckets
aws s3 ls

# List objects (recursive)
aws s3 ls s3://my-bucket/prefix/ --recursive --human-readable

# Delete bucket (must be empty)
aws s3api delete-bucket --bucket my-bucket
# Force-delete including all objects and versions
aws s3 rb s3://my-bucket --force
```

### Object Operations

```bash
# Upload single file
aws s3 cp localfile.txt s3://my-bucket/path/file.txt

# Download single file
aws s3 cp s3://my-bucket/path/file.txt ./local/

# Sync directory (upload new/changed, delete removed)
aws s3 sync ./dist s3://my-bucket/dist/ --delete

# Sync with exclude/include patterns
aws s3 sync ./build s3://my-bucket/ \
  --exclude "*.log" --include "*.html" --include "*.js"

# Move (copy + delete)
aws s3 mv s3://my-bucket/old-path/ s3://my-bucket/new-path/ --recursive

# Delete object
aws s3 rm s3://my-bucket/path/file.txt
# Delete prefix recursively
aws s3 rm s3://my-bucket/prefix/ --recursive
```

### Presigned URLs

```bash
# Generate presigned GET URL (default 3600s)
aws s3 presign s3://my-bucket/file.zip

# Custom expiry (seconds)
aws s3 presign s3://my-bucket/file.zip --expires-in 86400

# Presigned PUT URL (for upload by third party) — requires s3api
aws s3api create-presigned-url \   # Note: not available in all CLI versions
  --bucket my-bucket --key uploads/file.txt --http-method PUT

# Alternative: generate PUT presigned URL via CLI
python3 -c "
import boto3, sys
s3 = boto3.client('s3')
url = s3.generate_presigned_url('put_object',
  Params={'Bucket':'my-bucket','Key':'uploads/file.txt'},
  ExpiresIn=3600)
print(url)
"
```

### Lifecycle Policies

```bash
# Apply lifecycle rule (expire objects after 30 days)
aws s3api put-bucket-lifecycle-configuration \
  --bucket my-bucket \
  --lifecycle-configuration '{
    "Rules": [{
      "ID": "expire-old-logs",
      "Status": "Enabled",
      "Filter": {"Prefix": "logs/"},
      "Expiration": {"Days": 30},
      "NoncurrentVersionExpiration": {"NoncurrentDays": 7}
    }]
  }'

# View current lifecycle rules
aws s3api get-bucket-lifecycle-configuration --bucket my-bucket

# Delete lifecycle configuration
aws s3api delete-bucket-lifecycle --bucket my-bucket
```

### Versioning

```bash
# Enable versioning
aws s3api put-bucket-versioning --bucket my-bucket \
  --versioning-configuration Status=Enabled

# List object versions
aws s3api list-object-versions --bucket my-bucket --prefix path/to/file.txt

# Delete a specific version
aws s3api delete-object --bucket my-bucket \
  --key path/to/file.txt --version-id <version-id>

# Restore deleted object (delete the delete marker)
aws s3api delete-object --bucket my-bucket \
  --key path/to/file.txt --version-id <delete-marker-id>
```

### Server-Side Encryption

```bash
# Enable default encryption (SSE-S3)
aws s3api put-bucket-encryption --bucket my-bucket \
  --server-side-encryption-configuration '{
    "Rules": [{"ApplyServerSideEncryptionByDefault": {"SSEAlgorithm": "AES256"}}]
  }'

# Enable SSE-KMS with specific key
aws s3api put-bucket-encryption --bucket my-bucket \
  --server-side-encryption-configuration '{
    "Rules": [{"ApplyServerSideEncryptionByDefault": {
      "SSEAlgorithm": "aws:kms",
      "KMSMasterKeyID": "arn:aws:kms:us-east-1:123456789012:key/key-id"
    }}]
  }'

# Upload with explicit encryption
aws s3 cp file.txt s3://my-bucket/ --sse aws:kms --sse-kms-key-id alias/my-key
```

### Cross-Account Access

```bash
# Bucket policy allowing another account to read
aws s3api put-bucket-policy --bucket my-bucket --policy '{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",
    "Principal": {"AWS": "arn:aws:iam::OTHER_ACCOUNT_ID:root"},
    "Action": ["s3:GetObject", "s3:ListBucket"],
    "Resource": [
      "arn:aws:s3:::my-bucket",
      "arn:aws:s3:::my-bucket/*"
    ]
  }]
}'

# Block public access (recommended on all buckets)
aws s3api put-public-access-block --bucket my-bucket \
  --public-access-block-configuration \
  BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true
```

## Patterns

### Static Website Hosting

```bash
aws s3 website s3://my-bucket/ --index-document index.html --error-document error.html
aws s3api put-bucket-policy --bucket my-bucket --policy '{...public read policy...}'
```

### Large File Multipart Upload

```bash
# aws s3 cp handles multipart automatically for files > 8MB
# Force multipart with custom part size
aws s3 cp largefile.iso s3://my-bucket/ --multipart-threshold 64MB --multipart-chunksize 64MB
```

## Gotchas

- **Bucket names are globally unique across all AWS accounts** — choose a prefix like `{org}-{env}-{purpose}` to avoid conflicts
- **Versioning cannot be fully disabled** — once enabled, versioning can be suspended but not turned off; old versions remain and incur storage costs
- **S3 is eventually consistent for overwrite PUTs in some regions** — for strong consistency use `s3:GetObjectAttributes` after write to verify
- **`--delete` flag in sync is destructive** — files absent from source will be deleted from destination; test with `--dryrun` first
- **Cross-region replication requires versioning on both buckets** — set up replication rules after enabling versioning on source and destination
- **Access denied on `s3 ls`** — if the IAM policy has `s3:GetObject` but not `s3:ListBucket`, listing fails silently; both are needed for most CLI workflows
